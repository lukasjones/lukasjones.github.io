What is SQL you ask?

SQL stands for Structure Query Language. It's used to ask and tell the database to show and do stuff.

Below are a list of cheats (query's(commands)) and their outputs. Commands/querys/cheats are written in upper case. This is not required when querying the database but is done a lot to differentiate commands from other pieces of the statemtent.

grocery_list
id list_name total_cost total_weight completed?
1 Safeway 56.98 20 no
2 Trader Joes 25.00 7 no
3 Whole Foods 299.99 56 yes
4 Costco 200.00 50 no

^^ This table will be referenced in the cheats so study up!

SELECT and FROM

SELECT is used in the majority of querys to ask for information FROM a table. Alternatively, it can also be used to SELECT and change something in a table(more on that later).

SELECT * FROM table_name;
// This code selects all the tables from the specified table_name.
SELECT column_name_1, column_name_2 FROM table_name;
// You can select specific columns by using the name of the columns.
SELECT * FROM grocery_list;
// Ouput:
id list_name total_cost total_weight completed?
1 Safeway 56.98 20 no
2 Trader Joes 25.00 7 no
3 Whole Foods 299.99 56 yes
4 Costco 200.00 50 no

SELECT id, list_name FROM grocery_list;
// Ouput:
id list_name
1 Safeway
2 Trader Joes
3 Whole Foods
4 Costco

WHERE

WHERE is used to query a specific row in a table.

SELECT * FROM table_name WHERE column_name (operator) value;
// This will select all the columns where a specific column_name is a specific value. Basically it gives you the row or rows where that column_name = the value.
// For example.
SELECT * FROM grocery_list WHERE id = 3;
// Output:
id list_name total_cost total_weight completed?
3 Whole Foods 299.99 56 yes

AND & OR

AND and OR are used in SQL in conjunction with WHERE to limit or open the search. It's just like the Ruby && and ||.

SELECT * FROM table_name WHERE column_name_1 = value AND column_name_2 = value;
// This will select all the columns where a column_name_1 = the value specified AND column_name_2 = the value specified. If they are not both true, the row will not be selected.
SELECT * FROM grocery_list WHERE ID = 3 AND completed = 'yes';
// Output:
id list_name total_cost total_weight completed?
3 Whole Foods 299.99 56 yes

SELECT * FROM table_name WHERE column_name_1 = value OR column_name_2 = value;
// This will select all the columns where a column_name_1 = the value or column_name_2 = the value. Basically it's selecting the rows that return true for both the WHERE statements.
SELECT * FROM grocery_list WHERE id = 2 OR id = 4;
// Output:
id list_name total_cost total_weight completed?
2 Trader Joes 25.00 7 no
4 Costco 200.00 50 no

ORDER BY

ORDER BY is used to order your output from a SELECT statement. You can organize items in ASCending and DESCending order.

SELECT * FROM table_name ORDER BY column_name (ASC|DESC);
// This will select all the columns from the table_name and sort them by the column_name specified in either ASCending or DESCending order depending on what you specify. You can specify two column_names where you organize by the first column_name and then if there are many duplicates of the first column_name, you order by the second_column name.
SELECT * FROM grocery_list ORDER BY id DESC;
// Output:

id list_name total_cost total_weight completed?
4 Costco 200.00 50 no
3 Whole Foods 299.99 56 yes
2 Trader Joes 25.00 7 no
1 Safeway 56.98 20 no

INSERT INTO

INSERT INTO is a way of inserting data into your tables. It uses two key word sets "INSERT INTO" and "VALUES".

INSERT INTO table_name VALUES (value1, value2, value3...);
// This is selecting a table and putting the values you want. You must make sure that you have the same number of values as columns in your table.
INSERT INTO table_name (column1, column2, column3...) VALUES (value1, value2, value3...);
// This is selecting the table_name and then the column names you want to insert your values into. Doesn't have to be all the columns.
INSERT INTO grocery_list VALUES (5, "Grocery Outlet", 46.50, 20, yes);
INSERT INTO grocery_list (id, list_name, total_cost, total_weight, completed?) VALUES(5, "Grocery Outlet", 46.50, 20, yes);
// Output:
(You have made changes to the database. Rows affected: 1)
// Table now looks like:
id list_name total_cost total_weight completed?
1 Safeway 56.98 20 no
2 Trader Joes 25.00 7 no
3 Whole Foods 299.99 56 yes
4 Costco 200.00 50 no
5 Grocery Outlet 46.50 20 yes

UPDATE

UPDATE is used to alter existing values in a table. It uses three keywords: UPDATE, SET and WHERE to specify the new values and which values you will be replacing.

UPDATE table_name SET column1=value1,column2=value2,... WHERE column=value;
// This is saying in the table_name set column1 equal to value 1 etc. WHERE the column is eqal to a certain value. The last part selects the row or rows and the SET part changes the values for the specified columns.
UPDATE grocery_list SET list_name = "Tokyo Fish Market", completed? = "yes" WHERE id = 1;
// Output: (You have made changes to the database. Rows affected: 1)
// Table now looks like:
id list_name total_cost total_weight completed?
1 Tokyo Fish Market 56.98 20 yes
2 Trader Joes 25.00 7 no
3 Whole Foods 299.99 56 yes
4 Costco 200.00 50 no

DELETE

DELETE is used to delete selected row(s) from a table.

DELETE FROM table_name WHERE column_name = value;
// This deletes the rows in table_name where column_name equals a certain value.
DELETE FROM grocery_list WHERE id = 2;
// Output: (You have made changes to the database.)
// Table now looks like:
id list_name total_cost total_weight completed?
1 Safeway 56.98 20 no
3 Whole Foods 299.99 56 yes
4 Costco 200.00 50 no

SELECT TOP

SELECT TOP selects the top (number that you specify) rows from a table you specify. The number can be an integer or percentage. percentage syntax is for ex. "50 PERCENT".

SELECT TOP (number) FROM (optional: column_name) table_name;
// This selects the top (number) values from the specified table.
SELECT TOP 2 FROM grocery_list;
// Output:
id list_name total_cost total_weight completed?
1 Safeway 56.98 20 no
2 Trader Joes 25.00 7 no
